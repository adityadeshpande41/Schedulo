Build only the FRONTEND for an AI scheduling product called “Schedulo”.

Important:
- FRONTEND ONLY for this version
- Do NOT build the backend yet
- Do NOT integrate real calendar APIs yet
- Use realistic mock data for now
- However, structure the frontend cleanly so it can later connect to a Python backend

- Individual backend agents will later be implemented in Python

Product concept:
Schedulo is an AI-powered multi-agent meeting scheduler. Different agents work together to find the best meeting times based on calendar availability, user behavior, scheduling preferences, negotiation across attendees, and orchestration of final decisions.

For this frontend version, visually communicate that these agents exist and are working behind the scenes, but use mock data and simulated agent activity.

Future backend architecture to design for:
- Python backend
- FastAPI APIs
- Separate Python agents/modules such as:
  - calendar agent
  - behavior / preference agent
  - coordination / negotiation agent
  - orchestrator
The frontend should be organized so it can later connect to endpoints like:
- /schedule/request
- /slots/recommendations
- /agents/activity
- /preferences/:id
- /meetings/:id/decision
